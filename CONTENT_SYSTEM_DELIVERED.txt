QUICK START (5 MINUTES)
================================================================================

1. Install dependencies:
   cd agents/content_agents
   pip install -r requirements.txt

2. Start agents (3 terminals):
   Terminal 1: python researcher_agent.py
   Terminal 2: python creator_agent.py
   Terminal 3: python scheduler_agent.py

3. Send test request:
   python << 'EOF'
   import pika, json, uuid
   conn = pika.BlockingConnection(pika.ConnectionParameters('localhost'))
   ch = conn.channel()
   ch.exchange_declare(exchange='content', exchange_type='topic', durable=True)
   msg = {
       'correlation_id': str(uuid.uuid4()),
       'topic': 'AI in Healthcare',
       'keywords': ['AI', 'healthcare'],
       'depth': 'comprehensive'
   }
   ch.basic_publish(exchange='content', routing_key='content.research.request',
                   body=json.dumps(msg),
                   properties=pika.BasicProperties(delivery_mode=2))
   print("✓ Request sent!")
   conn.close()
   EOF

4. Check results in Terminal 1

GEMINI API (Free Tier):
  • Researcher: ~500 tokens per research
  • Editor: ~300 tokens per outline
  • Writer: ~1000 tokens per article
  • Strategist: ~400 tokens per plan
  • Free tier: 60 requests/minute
  • Cost: FREE (up to 15,000 requests/month)

OPENROUTER (Grok 4.1):
  • Creator: ~200 tokens per tweet set
  • Cost: ~$0.01-0.05 per request
  • Rate limit: 100 requests/minute
  • Monthly cost: ~$5-10 (500 requests)

TWITTER API v2:
  • Scheduler: 1 request per tweet
  • Free tier: 450 requests/15 minutes
  • Cost: FREE

RABBITMQ:
  • Messaging broker
  • Self-hosted (free)
  • Unlimited messages

TOTAL MONTHLY COST: ~$5-10 (mostly OpenRouter)

TROUBLESHOOTING
================================================================================

ISSUE: ModuleNotFoundError: No module named 'pika'

ISSUE: Connection refused
FIX: Ensure RabbitMQ is running
     docker-compose up rabbitmq

ISSUE: API key not valid
FIX: Check .env, regenerate key in API console

ISSUE: Twitter auth failed

     Check token expiration
     Regenerate tokens if needed

ISSUE: Timeout
FIX: Increase timeout in agent code
     Check API status
     Reduce prompt complexity

ISSUE: Rate limit exceeded
FIX: Implement exponential backoff
     Reduce request frequency
     Use queue management

================================================================================
DOCKER DEPLOYMENT
================================================================================

ADD TO docker-compose.yml:

  researcher-agent:
    build:
      context: ./agents/content_agents
      dockerfile: Dockerfile
    environment:
      GEMINI_API_KEY: ${GEMINI_API_KEY}
      RABBITMQ_HOST: rabbitmq
    depends_on:
      rabbitmq:
        condition: service_healthy
    networks:
      - automation
    restart: unless-stopped
    command: python -u researcher_agent.py

  creator-agent:
    build:
      context: ./agents/content_agents
      dockerfile: Dockerfile
    environment:
      OPENROUTER_API_KEY: ${OPENROUTER_API_KEY}
      RABBITMQ_HOST: rabbitmq
    depends_on:
      rabbitmq:
        condition: service_healthy
    networks:
      - automation
    restart: unless-stopped
    command: python -u creator_agent.py

  scheduler-agent:
    build:
      context: ./agents/content_agents
      dockerfile: Dockerfile
    environment:
      TWITTER_API_KEY: ${TWITTER_API_KEY}
      TWITTER_API_SECRET: ${TWITTER_API_SECRET}
      TWITTER_BEARER_TOKEN: ${TWITTER_BEARER_TOKEN}
      TWITTER_ACCESS_TOKEN: ${TWITTER_ACCESS_TOKEN}
      TWITTER_ACCESS_TOKEN_SECRET: ${TWITTER_ACCESS_TOKEN_SECRET}
      RABBITMQ_HOST: rabbitmq
    depends_on:
      rabbitmq:
        condition: service_healthy
    networks:
      - automation
    restart: unless-stopped
    command: python -u scheduler_agent.py

THEN:
  docker-compose up researcher-agent creator-agent scheduler-agent
